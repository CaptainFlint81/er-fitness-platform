# E.R. Fitness Deployment Checklist

## Pre-Deployment Verification

- Production build passes with `npm run build`.
- TypeScript passes with `npm run typecheck`.
- Lint passes with `npm run lint`.
- Legal routes are available: `/legal`, `/legal/privacy`, `/legal/terms`, `/legal/disclaimer`.
- Protected education routes link to the disclaimer flow.
- SEO metadata exists for the primary route groups.
- Language support is available through locale-aware labels and selectors.
- Community, resource library, source directory, education, exercise, nutrition, recovery, and media routes are included in the deployment surface.
- `sitemap.xml` and `robots.txt` are generated by Next.js metadata routes.

## Vercel Deployment Steps

1. Push `er-fitness-platform` to the connected Git provider.
2. Open Vercel and choose `Add New...` then `Project`.
3. Import the repository.
4. Keep the framework preset as `Next.js`.
5. Use `npm install` for install command unless the Vercel import auto-detects it.
6. Use `npm run build` for the build command.
7. Keep the output directory as the Next.js default.
8. Add production environment variables:
   - `NEXT_PUBLIC_SITE_URL=https://www.<your-purchased-domain>`
   - `NEXT_PUBLIC_SUPABASE_URL=`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY=`
   - `SUPABASE_SERVICE_ROLE_KEY=` only when server-side Supabase workflows are implemented.
   - `NEXT_PUBLIC_ANALYTICS_PROVIDER=` when analytics are approved.
   - `NEXT_PUBLIC_ANALYTICS_ID=` when analytics are approved.
9. Deploy.
10. After deployment, open the Vercel production URL and complete the post-deploy smoke test below.

## Domain and DNS Steps

1. Decide the canonical production domain before adding DNS records.
   - Recommended canonical domain: `https://www.<your-purchased-domain>`.
   - Apex domain: `<your-purchased-domain>`.
   - `www` subdomain: `www.<your-purchased-domain>`.
   - Set `NEXT_PUBLIC_SITE_URL` to the canonical URL exactly.
2. In the Vercel project, open `Settings` then `Domains`.
3. Add the apex domain: `<your-purchased-domain>`.
4. Add the `www` subdomain: `www.<your-purchased-domain>`.
5. In Vercel, choose the canonical production domain and redirect the non-canonical domain to it.
6. At the domain registrar, set the apex record exactly as Vercel shows it:
   - Host: `@`
   - Type: use the record type Vercel displays for the apex domain.
   - Value: copy the Vercel-provided value from the project domain screen.
7. At the domain registrar, set the `www` record exactly as Vercel shows it:
   - Host: `www`
   - Type: use the record type Vercel displays for the `www` subdomain.
   - Value: copy the Vercel-provided value from the project domain screen.
8. Do not hardcode DNS values from old guides, screenshots, or memory. Use the values Vercel shows for this exact project.
9. Remove conflicting `A`, `AAAA`, `CNAME`, or forwarding records for the same hosts.
10. Wait for DNS propagation.
11. Confirm Vercel shows both domains as valid.
12. Confirm HTTPS is active for the canonical domain and the redirected domain.
13. Re-run the smoke test using the final canonical domain.

## Deployment File Checks

- `DEPLOYMENT_CHECKLIST.md` contains Vercel import, build, domain, DNS, HTTPS, env var, and smoke-test steps.
- `.env.example` includes `NEXT_PUBLIC_SITE_URL`, Supabase placeholders, and analytics placeholders.
- `src/app/robots.ts` generates `robots.txt` from the configured site URL.
- `src/app/sitemap.ts` generates static and dynamic education/community sitemap entries.
- `src/lib/seo.ts` centralizes metadata base URL, route metadata, Open Graph, and social sharing defaults.
- Legal routes exist for privacy, terms, disclaimer, and the legal index.
- Disclaimer gate stores local acknowledgment and routes users to legal pages.
- Localization routing and language selector are present in the header/footer flow.

## Remaining Launch Items

- Replace `<your-purchased-domain>` with the actual purchased domain.
- Choose whether analytics are approved and which provider to use.
- Add real Supabase values only when backend workflows are turned on.
- Complete professional review for injury, adaptive fitness, youth fitness, women's fitness, nutrition, recovery, and medical-adjacent content before presenting it as reviewed guidance.
